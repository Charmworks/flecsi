Definition: field_interface.h:78

This macro registers field data with a topology type. Data registration creates a data attribute for the given topology type. This call does not necessarily cause memory to be allocated. Its primary function is to describe the field data to the runtime. Memory allocation will likely be deferred, and will only be done if the field is mapped into a task.

Parameters
topology_typeThe topology type.
nspaceThe string namespace to use to register the variable.
nameThe string name of the data variable to register.
data_typeThe data type to store, e.g., double or my_type_t.
storage_classThe storage type for the data storage_class_t.
versionsThe number of versions of the data to register. This parameter can be used to manage multiple data versions, e.g., for new and old state.

template<typename DATA_POLICY >
template<typename TOPOLOGY_TYPE , size_t STORAGE_CLASS, typename DATA_TYPE , size_t NAMESPACE, size_t NAME, size_t VERSIONS, size_t INDEX_SPACE = 0>
static bool flecsi::data::field_interface_u< DATA_POLICY >::add_field ( std::string const &  name)
inlinestatic

Add a field to the given topology type. This method should be thought of as registering a field attribute on the topology. All instances of the topology type will have this attribute. However, this does not mean that each topology instance will allocate memory for this attribute. Attribute instances will be created only when they are actually mapped into a task.

Template Parameters
TOPOLOGY_TYPEThe data client type on which the data attribute should be registered.
STORAGE_CLASSThe storage type for the data attribute.
DATA_TYPEThe data type, e.g., double. This may be P.O.D. or a user-defined type that is trivially-copyable.
NAMESPACEThe namespace key. Namespaces allow separation of attribute names to avoid collisions.
NAMEThe attribute name.
VERSIONSThe number of versions that shall be associated with this attribute.
INDEX_SPACEThe index space identifier.
Parameters
nameThe string version of the field name.
