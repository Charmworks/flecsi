Design Pattern: Tuple Walker

A tuple is a fix-sized collection of hetergeneous values. C++ has had support for tuples since C++11 through the std::tuple type. In addition to the tuple type itself, C++ provides utilities for creating and accessing tuples:

1 // Make a tuple of type <double, std::string, int>.
2 auto t = std::make_tuple(1.0, "a string", 10);
3 
4 // Get the zero'th element of the tuple 't'.
5 auto e = std::get<0>(t);
6 
7 // Get the type of the zero'th element.
8 using type = typename std::tuple_element<0, decltype(t)>::type;

One common use for tuples is to capture type and runtime information from a variadic argument list:

1 template<
2  size_t KEY,
3  typename ARG_TUPLE,
4  typename ... ARGS
5 >
6 static
7 decltype(auto)
8 execute_task(
9  launch_type_t launch,
10  size_t parent,
11  ARGS && ... args
12 )
13 {
14  // Create a tuple of the arguments to a user task.
15  ARG_TUPLE task_args = std::make_tuple(args ...);
16 } // execute_task

In the above example, we create a tuple instance from a variadic list of arguments that are to be passed to a user-defined task (FleCSI infers the tuple type when the task is registered.) As an example, consider the following:

1 double task(int a, double x, double y) {
2  if(a > 0) {
3  return x*y;
4  }
5  else {
6  return 0.0;
7  } // if
8 } // task
9 
10 flecsi_register_task(task, loc, single);
11 
12 int main() {
13  flecsi_execute_task(task, single, 1, 2.0, 3.0);
14 } // main

In this case, the variadic argument list is <int, double, double>. Suppose that we need to perform additional checks or processing on a particular argument type, e.g., the doubles must be non-negative. We can do this by applying a handle function to the argument tuple as it passes through the runtime:

1 init_args_t init_handles;
2 init_args.walk(args);
