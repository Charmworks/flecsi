68 // Use thread local storage for legion state information. The state_
69 // is set for each legion task invocation using the task name hash
70 // as a key. This seems like it should be safe, since multiple concurrent
71 // invocations of the same task can only occur on seperate threads.

73  std::stack<std::shared_ptr<legion_runtime_state_t>>> state_;
